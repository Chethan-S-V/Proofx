# ProofX Engineering Rules

Project: ProofX

Company: Wild En Tree (WET)

Core philosophy:

Trust what users prove, not what they claim.

Rules:

* Never regenerate existing files.
* Never rewrite working code.
* Use TypeScript strict mode.
* Use reusable services.
* Keep components small.
* Separate UI and business logic.
* Use Zod validation.
* Use Drizzle ORM.
* Use Supabase Auth.
* Use PostgreSQL.
* Use App Router.
* Use server components by default.
* Avoid unnecessary dependencies.
* Avoid microservices.
* Avoid Kubernetes.
* Build incrementally.

When implementing a feature:

1. Analyze existing code.
2. List files to create.
3. List files to modify.
4. Generate code.
5. Verify imports.
6. Verify TypeScript compatibility.
7. Verify build success.

Never proceed to another phase without completing the current phase.
